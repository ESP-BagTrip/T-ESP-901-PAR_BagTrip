# Internationalisation (i18n)

> Derniere mise a jour : 2026-03-26

## Vue d'ensemble

BagTrip supporte deux langues : francais (FR, langue par defaut) et anglais (EN). L'internationalisation repose sur le systeme standard Flutter `gen_l10n` avec des fichiers ARB comme source de verite. Les fichiers generes (`app_localizations.dart`, `app_localizations_en.dart`, `app_localizations_fr.dart`) fournissent un acces type-safe aux chaines via `AppLocalizations.of(context)!.keyName`.

## Configuration

### l10n.yaml

**Fichier** : `bagtrip/l10n.yaml`

```yaml
arb-dir: lib/l10n
template-arb-file: app_fr.arb
output-localization-file: app_localizations.dart
```

- Le template est `app_fr.arb` (francais = langue de reference)
- Les fichiers generes sont dans `bagtrip/lib/l10n/`

### Integration dans MaterialApp

**Fichier** : `bagtrip/lib/main.dart`

```dart
MaterialApp.router(
  localizationsDelegates: AppLocalizations.localizationsDelegates,
  supportedLocales: AppLocalizations.supportedLocales,
  // ...
)
```

## Fichiers ARB

| Fichier | Lignes | Role |
|---------|--------|------|
| `bagtrip/lib/l10n/app_fr.arb` | 1275 | Template de reference (francais) |
| `bagtrip/lib/l10n/app_en.arb` | 1275 | Traduction anglaise |

Les deux fichiers ont le meme nombre de lignes, indiquant une parite complete des cles.

### Structure d'une entree ARB

**Chaine simple** :
```json
"searchFlightButton": "Rechercher votre vol"
```

**Chaine avec placeholder** :
```json
"multiDestFlightTitle": "Vol {index}",
"@multiDestFlightTitle": {
  "placeholders": {
    "index": { "type": "int" }
  }
}
```

**Chaine avec pluriel (ICU)** :
```json
"stopsLabel": "{count, plural, =0{Vol direct} =1{1 escale} other{{count} escales}}",
"@stopsLabel": {
  "placeholders": {
    "count": { "type": "int" }
  }
}
```

## Fichiers generes

| Fichier | Role |
|---------|------|
| `bagtrip/lib/l10n/app_localizations.dart` | Classe abstraite `AppLocalizations` + delegate + liste des locales |
| `bagtrip/lib/l10n/app_localizations_fr.dart` | Implementation francaise |
| `bagtrip/lib/l10n/app_localizations_en.dart` | Implementation anglaise |

Ces fichiers sont regeneres automatiquement par `flutter gen-l10n` (ou `flutter pub run build_runner build`).

## Utilisation dans le code

### Acces aux chaines

```dart
final l10n = AppLocalizations.of(context)!;

// Chaine simple
Text(l10n.searchFlightButton)

// Avec placeholder
Text(l10n.multiDestFlightTitle(3))  // "Vol 3" / "Flight 3"

// Avec pluriel
Text(l10n.stopsLabel(2))  // "2 escales" / "2 stops"
```

### Exemples concrets du projet

**OfflineBanner** (`bagtrip/lib/components/offline_banner.dart`) :
```dart
AppLocalizations.of(context)!.offlineMode
// FR: "Vous etes hors ligne. Donnees en cache affichees."
// EN: "You are offline. Showing cached data."
```

**ActivityCard** (`bagtrip/lib/activities/widgets/activity_card.dart`) :
```dart
l10n.activityCardSemanticLabel(
  activity.title,
  semanticTime,
  activity.location ?? '',
  semanticStatus,
)
```

**BottomTabBar** (`bagtrip/lib/components/bottom_tab_bar.dart`) :
```dart
l10n.tabActivityWithBadge(activityBadgeCount)
// FR: "Activite, {count} notifications"
```

## Changement de langue

### SettingsBloc

**Fichier** : `bagtrip/lib/settings/bloc/settings_bloc.dart`

Le `SettingsBloc` gere le choix de langue via l'evenement `ChangeLanguage` :

```dart
class ChangeLanguage extends SettingsEvent {
  final String language;
  ChangeLanguage(this.language);
}
```

L'etat par defaut est `selectedLanguage = 'Français'`.

**Note** : Le `SettingsBloc` stocke la langue comme chaine de caracteres (`'Français'`, `'English'`) mais le `MaterialApp` ne connecte pas encore cette valeur a la propriete `locale`. Le changement de langue via le bloc n'a donc pas d'effet sur les traductions affichees — la locale suit le systeme.

### UI de selection

**Fichier** : `bagtrip/lib/profile/widgets/preferences_section.dart`

La section Preferences du profil affiche un selecteur de langue et un selecteur de theme, connectes au `SettingsBloc`.

## Workflow d'ajout d'une cle

1. Ajouter la cle dans `app_fr.arb` (template)
2. Ajouter la traduction dans `app_en.arb`
3. Si placeholder/pluriel : ajouter le bloc `@keyName` avec les metadonnees
4. Regenerer : `flutter gen-l10n` ou `flutter pub run build_runner build`
5. Utiliser : `AppLocalizations.of(context)!.newKey`

## Tests

Les tests utilisent systematiquement `AppLocalizations.localizationsDelegates` et `AppLocalizations.supportedLocales` dans les wrappers de test :

```dart
// test/accessibility/a11y_test_helpers.dart
Widget buildTestableWidget(Widget child, {double textScale = 1.0}) {
  return MaterialApp(
    localizationsDelegates: AppLocalizations.localizationsDelegates,
    supportedLocales: AppLocalizations.supportedLocales,
    locale: const Locale('en'),
    // ...
  );
}
```

Le `SettingsBloc` est teste dans `test/blocs/settings_bloc_test.dart` (changement de theme et de langue).

## Ce qu'il manque

| Element | Description | Priorite |
|---------|-------------|----------|
| Locale effective non connectee | `SettingsBloc.selectedLanguage` n'est pas mappe a `MaterialApp.locale`. Le changement de langue dans les preferences n'a aucun effet reel. (`bagtrip/lib/main.dart`, ligne ~185) | P0 |
| Persistance de la langue | Le choix de langue n'est pas persiste (pas de SharedPreferences/Hive). Il est perdu au redemarrage. (`bagtrip/lib/settings/bloc/settings_bloc.dart`) | P1 |
| Tests de parite ARB | Pas de test automatise verifiant que `app_en.arb` et `app_fr.arb` ont exactement les memes cles. Un oubli de traduction passe silencieusement. | P1 |
| Troisieme locale | Aucune autre langue que FR et EN n'est supportee. Pour un marche international, l'ajout d'ES/DE/IT serait pertinent. | P2 |
| Formatage des dates localise | Les dates dans certaines vues utilisent `DateFormat` avec des patterns fixes au lieu de `DateFormat.yMd(locale)`. | P2 |
| Tests widget en FR | Tous les tests widget utilisent `locale: const Locale('en')`. Aucun test ne verifie le rendu en francais. | P2 |
